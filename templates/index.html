Optimal Placements of Speed Feedback Signs Throughout Boston

Enter parameters and click submit to see the resulting map

Mean Skew 1 We filter intersections by being below the mean distance to closest accident cluster or not, so skewing this value allows a greater or smaller number of intersections to be considered as candidates. Above 1 equates to fewer candidates, below 1 equates to more candidates.
Radius 2.0 The radius in miles from the Boston Common where we will consider placing speed feedback signs.
Cluster Divisors 15 When clustering accidents, the cluster count is determined by dividing the total number of accidents by this value.
Sign Count 30 The number of speed feedback signs to be placed by the algorith. Note: not all centroids will be able to be mapped to candidate intersections, resulting in a sign placement, if the buffer size is too large.
Buffer Size .5 The radius around a placed speed feedback sign in which another speed feedback sign cannot be placed.

Requests will take about 90 seconds on average for a never-before-seen set of parameters, but seen requests will be returned in approximately 10 seconds.
Invalid responses from our datasets may require individual scripts to repeat, taking additional time.
